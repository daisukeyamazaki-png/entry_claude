Chapter 1

Claudeの基礎知識

まずClaude全体の仕組みと、2つのツールがどう位置づけられるかを理解します。土台を固めることで、後の学習がスムーズになります。

1
Claudeとは何か
▼

ClaudeはAnthropicが開発した大規模言語モデル（LLM）をベースにしたAIアシスタントです。「会話」という形式で自然言語の指示を理解し、テキスト生成・コード記述・データ分析・ファイル操作などを行います。

重要なのは、ClaudeはWebブラウザから使うclaude.aiだけではなく、複数のインターフェースで利用できるという点です。

インターフェース対象特徴
claude.ai（Web）一般ユーザーブラウザで使えるチャット。最も手軽。
Claude Code開発者ターミナルから使うCLI。コードベースを直接操作できる。
Claude Coworkビジネスパーソンデスクトップアプリ。ファイル操作・業務自動化に特化。
API開発者・企業自社サービスにClaudeを組み込む。
💡

ポイント：どのインターフェースを使っても、中身は同じClaudeモデルです。違いは「何ができるか・何にアクセスできるか」というツールの差です。

2
Claude Code vs Cowork — 何が違うのか
▼

2つのツールは「電車とバス」の関係に似ています。どちらも目的地（タスクの完了）に連れて行ってくれますが、得意なルートがまったく異なります。

💻 Claude Code🖥️ Claude Cowork
操作方法ターミナル（CLI）デスクトップアプリ（GUI）
主な対象開発者・エンジニアビジネスパーソン全般
得意なことコード生成・デバッグ・リファクタ・スクリプト実行ファイル処理・レポート作成・定期タスク・データ集計
コードの知識あると有利（必須ではない）不要
ファイルアクセスカレントディレクトリ以下選択したフォルダ
外部連携MCPプラグインMCPプラグイン・スケジューラ
ℹ️

MCP（Model Context Protocol）とは、ClaudeがSlack・Gmail・Notionなど外部ツールと連携するための拡張機能です。両ツール共通でプラグインとして追加できます。

Chapter 2

Claude Code 入門

ターミナルから呼び出すAIアシスタント。コードベース全体を把握した上でコーディング・デバッグ・リファクタリングをサポートします。

1
なぜClaude Codeが強力なのか
▼

通常のclaude.aiにコードを貼り付けて質問する方法との決定的な違いは、「プロジェクト全体を文脈として持てる」点です。

  • 起動時にカレントディレクトリのファイル一覧を自動で読み込む
  • 「このファイルを見て」と言わなくても、関連ファイルを自動で参照する
  • コードを書くだけでなく、実際にファイルを編集・作成・削除できる
  • ターミナルコマンドを実行して結果を確認した上で次の行動をとれる
  • GitのステータスやPRの差分を理解した上でコミットメッセージを書ける
💡

レシピ本（コードの知識）なしで料理（開発）を注文できる、専属シェフのようなイメージです。「どんな材料があるか」を自分で確認した上で、最適な料理を作ってくれます。

2
セットアップ手順
▼
1

Node.js v18以上を確認する

Claude CodeはNode.jsランタイム上で動作します。ターミナル（Mac）またはコマンドプロンプト / PowerShell（Windows）でバージョンを確認してください。

$ node --version
v20.11.0 ← v18以上であればOK
> node --version
v20.11.0 ← v18以上であればOK
⚠️

v18未満の場合は nodejs.org からLTS版をインストールしてください。Windowsの場合は .msi インストーラーが簡単です。

2

npmでインストールする

$ npm install -g @anthropic-ai/claude-code

permission エラーが出る場合（Mac）：

$ sudo npm install -g @anthropic-ai/claude-code

PowerShell を管理者として実行してからインストールしてください。

> npm install -g @anthropic-ai/claude-code
ℹ️

スタートメニューで「PowerShell」を右クリック→「管理者として実行」で起動できます。

3

初回起動とサインイン

claudeコマンドを実行するとブラウザが開きます。Anthropicアカウントでサインイン後、ターミナルに戻ると使い始められます。コマンドはMac・Windows共通です。

$ claude
Opening browser for authentication...
✓ Authenticated successfully
Welcome to Claude Code! Type your message below.
ℹ️

Windowsの場合： ブラウザが自動で開かない場合は、ターミナルに表示されたURLをコピーしてブラウザに貼り付けてください。

4

プロジェクトフォルダで起動する

必ず作業したいプロジェクトのフォルダ内でclaudeを起動してください。パスの書き方だけMac/Windowsで異なります。

$ cd /Users/yourname/projects/my-project
$ claude
✓ Context loaded: 47 files across 12 directories
> cd C:\Users\yourname\projects\my-project
> claude
✓ Context loaded: 47 files across 12 directories
💡

エクスプローラーでフォルダを開いてアドレスバーに cmd と入力してEnterを押すと、そのフォルダでコマンドプロンプトが開きます。

3
基本的な使い方 — 4つのパターン
▼

エラーメッセージをそのまま貼り付けるのが最速です。「このエラーを直して」だけで、関連ファイルを自動で調査して修正してくれます。

You: 以下のエラーが発生しています。修正してください。

TypeError: Cannot read properties of undefined (reading 'map')
at UserList (src/components/UserList.tsx:45:22)
at renderWithHooks (react-dom/cjs/react-dom.development.js:14985:18)

Claude: UserList.tsx を確認します...
原因: props.users が undefined のタイミングで map() が呼ばれています。
オプショナルチェーン（?.）とデフォルト値で修正します。

// Before
const items = props.users.map(u => ...)
// After
const items = (props.users ?? []).map(u => ...)
💡

スタックトレース（エラーの発生箇所の一覧）を含めると、より正確に問題を特定できます。

何を作りたいかを日本語で説明するだけでOKです。既存のコードスタイルや命名規則を自動で学習した上でコードを生成します。

You: CSVファイルを読み込んで売上データを月別・カテゴリ別に集計し、
Excelファイルとして出力するPythonスクリプトを作ってください。

- ライブラリ: pandas, openpyxl

Claude: analyze_sales.py を作成しました。
実行: python analyze_sales.py sales.csv
💡

入力・出力・使用ライブラリを明示すると、意図通りのコードが一発で出やすくなります。

引き継いだコードや、読み解けないコードを日本語で説明してもらえます。「このファイル全体」でも「この関数だけ」でも指定できます。

You: src/auth/middleware.js を見て、以下を日本語で説明してください。
1. このファイル全体の役割
2. JWT検証の仕組み（行番号付きで）
3. セキュリティ上の注意点があれば指摘して

Claude: middleware.js の解説
役割: リクエストのたびにJWTトークンを検証する認証ミドルウェアです。
処理フロー: Authorization ヘッダーからトークンを取り出し → ...

コードの品質改善を任せられます。リファクタリングの方針（DRY・SOLID・可読性向上など）を指定するとより的確な提案が返ってきます。

You: utils/helpers.js を以下の方針でリファクタリングしてください。
- DRY原則：重複している処理を共通関数に切り出す
- 可読性：関数名・変数名を意図が伝わる名前に変更
- 副作用：グローバル変数への依存を排除する
変更前後の差分がわかるようにコメントを残してください。

Chapter 3

Claude Cowork 入門

GUIで動くデスクトップアシスタント。コードの知識がなくてもファイルを渡して「やって」と言えば業務タスクを実行してくれます。

1
Coworkの強みを理解する
▼

claude.aiのWebチャットとCoworkの最大の違いは、「自分のPCのファイルに直接アクセスできる」点です。ファイルをアップロードする手間なく、フォルダをまるごと作業スペースとして指定できます。

  • 指定フォルダ内のファイルを自由に読み書き・整理できる
  • ExcelやWord・PDFなど複数ファイルを一括処理できる
  • 定期タスクをスケジュール設定して、自動実行できる
  • MCPプラグインでSlack・Gmail・Googleカレンダー等と連携できる
  • 処理結果をそのまま指定フォルダに保存できる
💡

ファイル整理・データ集計・レポート作成など、毎月同じ手作業を繰り返しているルーティン業務こそCoworkが最も効果を発揮する領域です。

2
初回セットアップ
▼
1

Claude for Desktopをインストール

Anthropicの公式サイト（claude.ai/download）からMac/Windowsアプリをダウンロード・インストールします。

2

Claude.aiアカウントでサインイン

アプリを起動してAnthropicアカウントにサインインします。Pro以上のプランが必要です。

3

Coworkモードを選択する

画面上部のモード切り替えから「Cowork」を選択します。通常のチャットよりも多くのツール（ファイル操作・スケジュール）が解放されます。

4

作業フォルダを選択する

「フォルダを選択」から、Claudeに作業させたいフォルダを指定します。これがClaudeの「作業デスク」になります。フォルダ内のファイルをClaudeが読み書きできるようになります。

⚠️

重要なファイルが入ったフォルダを指定する場合は、事前にバックアップを取っておきましょう。Claudeはファイルの削除・上書きも実行できます。

Chapter 4

プロンプトエンジニアリング

Claudeへの指示は「設計図」です。曖昧な設計図からは意図通りのものは作れません。5つのテクニックで出力品質を劇的に上げましょう。

1
テクニック①：出力形式を明示する
▼

「何を」だけでなく「どんな形で」出力してほしいかを常に書きましょう。フォーマット・長さ・トーンの3つが重要です。

❌ 曖昧
「レポートを作って」
✅ 具体的
「10月の売上データをもとにWord形式でレポートを作って。構成はKPIサマリー・前月比・来月の推奨アクション3点。役員向けの簡潔なビジネス文書で、全体800字以内。」
❌ 曖昧
「メールを書いて」
✅ 具体的
「新規BtoB顧客Aさんへの初回提案メールを件名＋本文で書いて。本文400字以内・ビジネス敬語・訴求点はサポート体制とオンボーディングの手厚さで。」
2
テクニック②：役割を与える
▼

プロンプトの冒頭で「あなたは〜です」と役割を定義すると、その専門家の視点から回答してくれます。専門的な出力が必要なときに特に効果的です。

# 役割を与えない場合
「競合分析をしてください」
→ 一般的な情報をまとめた回答になりがち

# 役割を与えた場合
「あなたはBtoB SaaS業界で10年の経験を持つ
戦略コンサルタントです。以下の競合分析をしてください...」
→ 業界固有の視点・フレームワークを使った深い分析になる
3
テクニック③：例を見せる（Few-shot）
▼

期待する出力の例を1〜3つ見せると、パターンを学習してくれます。フォーマットが決まっている繰り返し作業に特に有効です。

以下のフォーマットで、商品説明文を10件作成してください。

【例】
商品名: プレミアムコーヒーメーカー
説明: 豆の旨みを最大限に引き出す98℃抽出で、
カフェの味をそのまま自宅に。忙しい朝も1分でセット完了。

【処理対象リスト】
1. ワイヤレスイヤホン
2. スマートウォッチ
...
4
テクニック④：NG条件を伝える
▼

「〜はしないで」という除外条件を書くことで、不要な出力を防げます。Claudeは良かれと思って余計な説明を加えることがあるので、NGを明示するのは効果的です。

# よく使うNG条件の例
「説明は不要です。コードだけ出力してください。」
「箇条書きは使わず、流れるような文章で書いてください。」
「前置きなしに本文から書き始めてください。」
「既存のコードを削除・変更せず、追加のみ行ってください。」
「推測で補完せず、不明な点は質問してください。」
5
テクニック⑤：段階的に依頼する（Chain-of-thought）
▼

複雑なタスクを一度に全部頼むより、「まずアウトラインを作って → 確認 → 肉付け」と段階的に進める方が質が上がります。途中で方向修正もできます。

# Stage 1: 構成を確認してから進める
You: 製品ランディングページのコピーを書きたいです。
まず構成案（見出しのみ）を3パターン提案してください。

Claude: 【案A】共感型 / 【案B】課題解決型 / 【案C】実績訴求型 ...

# Stage 2: 選択して詳細化
You: 案Bで進めてください。各セクションのコピーを書いてください。

# Stage 3: 微調整
You: ファーストビューの見出しをもっとインパクトのある表現に変えてください。
候補を3つ出してください。
💡

一度のプロンプトに全てを詰め込もうとするのは「ゴールを指定して地図を渡さず旅に出す」ようなものです。段階的な対話がより良いアウトカムにつながります。

6
黄金フォーマット — テンプレートをそのまま使う
▼

5つのテクニックをまとめたテンプレートです。コピーして自分のタスクに合わせて書き換えてください。

## 役割（任意・専門性が必要なとき）
あなたは[役割・専門性]です。

## タスク（必須）
[具体的に何をしてほしいか]

## 入力情報（必須）

・[関連する制約・前提条件]

## 出力形式（必須）
・形式: [Wordファイル/Excel/Markdown/コードのみ etc.]
・長さ: [文字数・行数・ページ数]

## NG事項（任意）

Chapter 5

実践ワークフロー

Claude CodeとCoworkを組み合わせた、IT企業での実際の活用シナリオです。どちらをいつ使うかの判断基準も身につけましょう。

1
シナリオA：データパイプラインを構築して分析を自動化する
▼

マーケターがGA4データを毎月自動で取得・集計・レポートするパイプラインを作るシナリオです。

Claude Code
スクリプト作成 → CSVデータ生成 → Cowork
レポート作成 → Word出力
# Step1: Claude Code でGA4データ取得スクリプトを作る
（Claude Codeで）
GA4 API から先月分のセッション数・CVR・チャネル別データを取得して
monthly_data_YYYYMM.csv に保存するPythonスクリプトを作ってください。
認証はサービスアカウントJSONを使います。

# Step2: Cowork でレポートに仕上げる
（Coworkで）
monthly_data_202410.csv を読んで、経営層向けの月次レポートをWord形式で作って。
グラフ付きで、前月比・3ヶ月トレンドも含めること。
2
シナリオB：コードレビューとドキュメント整備
▼

既存コードのレビューをClaude Codeで行い、そのドキュメントをCoworkで整備するシナリオです。

# Step1: Claude Code でコードレビューを実施
（Claude Codeで）
src/ 以下の全ファイルをレビューして、以下の観点で問題点を指摘してください。
- セキュリティ上のリスク（SQLインジェクション・XSS・認証漏れ）
- パフォーマンスのボトルネック
- 保守性の低いコード（マジックナンバー・長すぎる関数）
結果を review_result.md に保存してください。

# Step2: Cowork でドキュメントを整備
（Coworkで）
review_result.md を読んで、エンジニア向けの「コーディング規約.docx」を作成して。
「やること」「やってはいけないこと」のコード例付きで整理してください。
3
使い分けの判断基準まとめ
▼
やりたいこと推奨ツール理由
コードを書く・直す💻 Claude Codeファイルを直接編集できる
コマンドを実行する💻 Claude Codeターミナルと統合されている
GitのPR・コミットを扱う💻 Claude Codegit操作をサポート
ExcelやWordを操作する🖥️ CoworkOffice形式の出力に最適化
定期タスクを自動化する🖥️ Coworkスケジューラ機能がある
複数ファイルをまとめて処理🖥️ Coworkフォルダ単位で指定できる
Slack・Gmail等と連携する🖥️ Cowork（またはCode）どちらもMCP対応
コードの知識なしで使いたい🖥️ CoworkGUIで直感操作できる

📝 プロンプト5原則

① 出力形式を明示
形式・長さ・トーンを必ず書く
② 役割を与える

③ 例を見せる

④ NGを伝える
「〜はしないで」で不要な出力を排除
⑤ 段階的に進める
アウトライン確認→承認→詳細化
